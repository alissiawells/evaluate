Описание формата данных для заполнения базы данных асессорского интерфейса

Нужно подготовить списки "объектов для оценки", "возможных ответов", "задач", "уже имеющиеся оценок" (скорее всего пустой список), а также список "типов задач"" для асессоров.

Каждый объект в любом списке, это словарь.

Поля оцениваемых объектов:

'id' - положительный id объекта
'title' - строчка (имя объекта)
'content' - строчка (большое текстовое содержание объекта)
'timestamp' - строчка, кодирующая время в формате ("%Y-%m-%dT%H:%M:%S+00:00")
'info_list' - список строчек (дополнительные текстовое атрибуты)


Поля возможных ответов такие же как и у оцениваемых объектов. Если "возможные ответы" такие же как "оцениваемые объекты", то список возможных ответов можно оставить пустым


Поля типов задач:

'id' : положительный id типа задачи
'name' : строчка, название задачи
'instruction' : текст, содержание инструкции (можно в maкkdown формате)
'choices' : список пар строчек: (строка для хранения в БД, строка для отображения). По умолчанию используется такой список: [(u"-1",u'?'), (u"0", u"-"), (u"1", u"+/-"), (u"2", u"+")]
'default_choice' : строка из множества возможных ответов (для хранения в БД), обозначающая ответ по умолчанию. Для предыдущего примера это u"-1"
'timestamp' :  строчка, кодирующая время в формате ("%Y-%m-%dT%H:%M:%S+00:00"),
'number_of_assessments' : положительное натуральное число, минимальное число оценок для одного объекта. Значение по умолчанию 1
'random_order' : булевская переменная, нужно ли подавать задания для оценки в случайном порядке, по умалчанию True
'show_text_answer_field' : булевская переменная, определяет нужно или нет вообще показывать поле для текстового ответа

Поля задач:

'id' - положительный id задачи
'task_type_id' - положительный id типа задачи
'eval_object_id' - id оцениваемого объекта
'text_answer' - boolean, нужно ли обязательно требовать от асессора ответа
'variants_id' - список id возможных ответов, с которыми надо сравнить оцениваемый объект


Поля оценок:

'id' - положительный id инструкции
'task_id' - id задачи, соответствующей оценке
'assessor' - строчка, соответствующаю асессору (логин пользователя)
'updated' - время последнего изменения ответа; строчка, кодирующая время в формате ("%Y-%m-%dT%H:%M:%S+00:00")
'created' - время ответа; строчка, кодирующая время в формате ("%Y-%m-%dT%H:%M:%S+00:00")
'variants_answers' - список результатов оценки, длины списка вариантов ()
'text_answer' - строчка, соответствующая текстому ответу


Все списки нужно положить в словарь c ключами: 'evaluated_objects', 'answers', 'tasks', 'task_types', 'evaluations'. Далее нужно сдампить этот словарь в json файл. Собственно, этот json файл и нужен, чтобы запонить базу данных моего вебсервера.
